## 백준 1149번 문제 RGB 거리

![problem](Python/CodingTest/image/Pasted%20image%2020240327231957.png)

### 풀이

DP를 사용해야 하는 문제다.

정답률 63%에 육박하는 비교적 쉬운 문제이고 실제로도 그렇다.

다만 문제를 푸는 와중에 탐욕 알고리즘의 오류가 발생한다고 착각해 시간을 필요이상으로 써버린 문제.

문제에서 요구하는 세 가지 조건 중 3번 항목이 1번과 2번 항목을 포함하는 내용이기 때문에 3번 항목만 유의하며 답안을 작성하면 된다.

풀이 과정은 각각 빨간색, 초록색, 파란색으로 시작하는 세 개의 출발지점을 통해 연산을 누적해가며 최솟값을 구하면 된다.

가령 R\[0\]번의 연산 이후엔 R\[1\]번 연산을 수행하고 해당 연산에선 파란색 또는 초록색 집만 칠할 수 있다.

이렇게 연산을 누계해서 최종적으로 R\[n-1\]과 G\[n-1\], B\[n-1\]중에서 최솟값을 구하면 된다.

내가 착각한 건 탐욕 알고리즘의 오류였다.

만약 1번째 연산에서 98 99 100중에서 빨간색 집을 98로 칠했을 때 2번째 연산에서 1 99 100이라는 선택지가 있을 때 빨간색을 이미 선택했기 때문에 강제적으로 99나 100중 하나의 값을 선택해야하고, 그렇다면 최종해가 최적해가 되지 않을수도 있겠다는 착각이었다.

그러나 현재 연산에서 이전 연산을 평가하므로 정상적으로 최솟값을 찾아가는 것을 확인할 수 있었다.


### 답안

```python
n = int(sys.stdin.readline().rstrip())  
rgb = [0] * n  
  
for i in range(n):  
    rgb[i] = list(map(int, sys.stdin.readline().rstrip().split()))  
  
for i in range(1, n):  
    rgb[i][0] = min(rgb[i-1][1], rgb[i-1][2]) + rgb[i][0]  
    rgb[i][1] = min(rgb[i-1][0], rgb[i-1][2]) + rgb[i][1]  
    rgb[i][2] = min(rgb[i-1][0], rgb[i-1][1]) + rgb[i][2]  
  
print(min(rgb[n-1][0], rgb[n-1][1], rgb[n-1][2]))
```
