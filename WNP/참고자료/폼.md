>[!info] 파라미터 사용 팁
>참고로 컨트롤러를 거치지 않고 위처럼 직접 호출할 경우 반환값은
>func로 넘겨준 파라미터의 prefix로 handle이 붙는다.
>```js
>frm.addParam("func", "Example"); // handleExample 호출
>function handleExample(data, status, jqXHR) {}
>```
>또한 컨트롤러에서 ResultInfo로 반환하면 data 하위에 count, msg가 담기지만 컨트롤러를 거치지 않고 query_id를 통해 요청을 날리면 data 하위에 바로 list 형태의 데이터가 담긴다.
>
> **요약**
>* controller 거치는 경우: `if(data.count > 0)` = 정상 응답
>* controller 거치지 않는 경우: `if(data\[0])` = 정상 응답 (데이터 있음)
