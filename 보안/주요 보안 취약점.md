## 1. 네트워크 및 애플리케이션 공격
**SQL Injection (SQL 삽입):** * **내용:** 입력창에 악의적인 SQL 구문을 주입해 DB를 조작하는 공격.
    
    - **대응:** **Prepared Statement(매개변수화 질의)** 사용, 입력값 검증.
        
- **XSS (Cross-Site Scripting):** * **내용:** 게시판 등에 악성 스크립트를 삽입해 사용자 브라우저에서 실행되게 하는 공격. (Stored, Reflected, DOM 기반 구분 필수)
    
    - **대응:** `< > &` 등의 특수문자를 HTML Entity로 치환(치환 코드 암기 필수).
        
- **CSRF (Cross-Site Request Forgery):**
    
    - **내용:** 사용자가 자신의 의지와 무관하게 공격자가 의도한 행위(비밀번호 변경 등)를 서버에 요청하게 만드는 공격.
        
    - **대응:** **CSRF 토큰** 사용, 재인증 요구.


## 2. 시스템 및 네트워크 보안
- **DoS / DDoS 공격:** * **SYN Flooding:** TCP의 3-Way Handshake 취약점을 이용해 서버의 자원을 고갈시킴. (대응: **Syncookies** 설정)
    
    - **Slowloris (Slow HTTP Header DoS):** HTTP 헤더를 끝내지 않고 천천히 보내 연결을 유지함.
        
- **버퍼 오버플로우 (Buffer Overflow):**
    
    - **내용:** 정해진 메모리 크기를 넘쳐나게 해서 반환 주소(Return Address)를 조작해 악성 코드를 실행함.
