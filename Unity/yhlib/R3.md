# R3

## 개요

R3는 Cysharp(neuecc)가 개발한 **차세대 Reactive Extensions** 라이브러리로, UniRx의 후속작이다. 10년 이상의 Rx 경험을 바탕으로 재설계되었다.

**핵심 변경점 (UniRx 대비):**
- 에러 발생 시 자동 구독 해제 제거 (`OnError` → `OnErrorResume`으로 변경, 구독 계속됨)
- `IScheduler` 제거 → `TimeProvider` + `FrameProvider`로 대체
- 프레임 기반 연산자 네이티브 지원
- 구독 누수 방지를 위한 `ObservableTracker` 내장
- `async/await`와의 자연스러운 통합

**참고:** GitHub: https://github.com/Cysharp/R3

## 기본 사용법

### ReactiveProperty

값을 보유하면서 변경을 통지한다. **중복 값은 자동으로 무시**된다.

```csharp
using R3;

public class Enemy
{
    public ReactiveProperty<long> CurrentHp { get; private set; }
    public ReactiveProperty<bool> IsDead { get; private set; }

    public Enemy(int initialHp)
    {
        CurrentHp = new ReactiveProperty<long>(initialHp);
        IsDead = CurrentHp.Select(x => x <= 0).ToReactiveProperty();
    }
}

// 사용
enemy.CurrentHp.Value -= 99;                              // 값 변경
enemy.CurrentHp.Subscribe(x => Debug.Log("HP:" + x));     // 변경 구독
enemy.IsDead.Where(x => x).Subscribe(_ => OnDeath());     // 파생 상태 구독
```

**ReadOnlyReactiveProperty:** 외부에 읽기 전용으로 노출:
```csharp
readonly ReactiveProperty<string> _name = new("");
public ReadOnlyReactiveProperty<string> Name => _name;
// 현재 값 접근: Name.CurrentValue (UniRx의 .Value와 다름!)
```

**SerializableReactiveProperty (Inspector용):**
```csharp
public class PlayerController : MonoBehaviour
{
    public SerializableReactiveProperty<int> hp;
    public SerializableReactiveProperty<float> speed;
}
```

### Subject 종류

| 타입 | 설명 |
|------|------|
| `Subject<T>` | 기본 이벤트 분배 (값 보유 없음) |
| `BehaviorSubject<T>` | 최신 값 1개 보유, 구독 시 즉시 발행 |
| `ReactiveProperty<T>` | BehaviorSubject + 중복 제거 + `.Value` setter |
| `ReplaySubject<T>` | 여러 값을 개수/시간 기준으로 보유 |

### Subscribe 패턴

```csharp
// 값만 처리
observable.Subscribe(x => Debug.Log(x));

// 값 + 완료 처리
observable.Subscribe(
    x => Debug.Log(x),
    result => Debug.Log(result.IsSuccess ? "완료" : $"실패: {result.Exception}")
);

// 값 + 에러 + 완료 (풀 핸들링)
observable.Subscribe(
    x => Debug.Log(x),
    ex => Debug.LogError(ex),           // OnErrorResume: 구독 계속됨!
    result => Debug.Log("스트림 종료")    // OnCompleted: 스트림 종료
);

// 비동기 처리
button.OnClickAsObservable()
    .SubscribeAwait(async (_, ct) =>
    {
        var result = await SomeAsyncOperation(ct);
        Debug.Log(result);
    }, AwaitOperation.Drop);  // 실행 중 새 클릭 무시
```

`AwaitOperation` 옵션:

| 옵션 | 설명 |
|------|------|
| `Sequential` | 큐에 쌓아서 순서대로 실행 |
| `Drop` | 비동기 실행 중이면 새 값 무시 |
| `Switch` | 이전 실행 취소 후 새 값으로 실행 |
| `Parallel` | 모든 값 즉시 병렬 실행 |

### 주요 연산자

#### 팩토리

```csharp
Observable.Return(value)                    // 값 하나 즉시 발행
Observable.Interval(TimeSpan.FromSeconds(1)) // 1초마다 발행
Observable.Timer(TimeSpan.FromSeconds(5))    // 5초 후 1회 발행
Observable.EveryUpdate()                     // 매 프레임 발행
Observable.EveryValueChanged(obj, x => x.hp) // 값 변화 감지
Observable.FromEvent(add, remove)            // 이벤트 → Observable
Observable.FromAsync(asyncFunc)              // 비동기 → Observable
```

#### 변환

```csharp
.Select(x => x * 2)                        // 값 변환
.SelectMany(x => otherObservable)           // 1:N 변환 후 병합
.Pairwise()                                 // 연속 값을 쌍으로 묶기
.Scan((acc, x) => acc + x)                  // 누적 연산
```

#### 필터링

```csharp
.Where(x => x > 0)                         // 조건 필터
.DistinctUntilChanged()                     // 연속 중복 제거
.Take(5)                                    // 5개만 취함
.Skip(3)                                    // 3개 건너뜀
.TakeUntil(otherObservable)                 // 다른 Observable까지
```

#### 시간/Rate 제한

```csharp
.Debounce(TimeSpan.FromMilliseconds(300))   // 입력 멈춘 후 300ms 뒤 전달 (구 Throttle)
.DebounceFrame(10)                           // 프레임 기반
.ThrottleFirst(TimeSpan.FromSeconds(1))      // 1초에 첫 값만
.ThrottleFirstFrame(30)                      // 30프레임에 첫 값만
.Delay(TimeSpan.FromSeconds(1))              // 1초 지연
.Chunk(TimeSpan.FromSeconds(1))              // 1초 단위로 묶음 (구 Buffer)
```

#### 결합

```csharp
Observable.Merge(obs1, obs2)                 // 병합
Observable.Concat(obs1, obs2)                // 순서대로 연결
Observable.CombineLatest(obs1, obs2, (a, b) => a + b)  // 최신 값 조합
.WithLatestFrom(other, (a, b) => ...)        // 주 스트림 + 보조 최신 값
.Prepend(initialValue)                       // 앞에 초기값 추가 (구 StartWith)
```

### Unity 통합 (R3.Unity)

R3.Unity를 설치하면 자동으로 초기화됨 (직접 호출 불필요).

#### uGUI 확장

```csharp
button.OnClickAsObservable().Subscribe(_ => Debug.Log("클릭!"));
slider.OnValueChangedAsObservable().Subscribe(v => Debug.Log(v));
toggle.OnValueChangedAsObservable().Subscribe(isOn => Debug.Log(isOn));
inputField.OnValueChangedAsObservable()
    .Debounce(TimeSpan.FromMilliseconds(500))
    .Subscribe(text => Search(text));

// Text에 바인딩
observable.SubscribeToText(myText);
```

#### MonoBehaviour 트리거 (R3.Triggers)

```csharp
using R3.Triggers;

this.OnCollisionEnterAsObservable().Subscribe(col => Debug.Log("충돌!"));
this.OnTriggerEnterAsObservable().Subscribe(other => Debug.Log("트리거!"));
```

#### TimeProvider / FrameProvider

```csharp
// TimeScale 무시
Observable.Interval(TimeSpan.FromSeconds(5), UnityTimeProvider.UpdateIgnoreTimeScale);

// FixedUpdate 루프
Observable.EveryUpdate(UnityFrameProvider.FixedUpdate);
```

### 구독 해제

#### 1. AddTo (GameObject 수명 연동) — 가장 간단

```csharp
Observable.EveryUpdate().Subscribe(x => Move()).AddTo(this);        // MonoBehaviour
Observable.EveryUpdate().Subscribe(x => Move()).AddTo(gameObject);  // GameObject
// → GameObject 파괴 시 자동 해제
```

#### 2. DisposableBag (구조체, 할당 없음)

```csharp
DisposableBag disposables;

void Start()
{
    Observable.EveryUpdate().Subscribe(_ => Move()).AddTo(ref disposables);
}

void OnDestroy()
{
    disposables.Dispose();
}
```

#### 3. CancellationToken으로 완료

```csharp
Observable.Interval(TimeSpan.FromSeconds(1), cancellationToken: destroyCancellationToken)
    .Subscribe(x => Debug.Log(x));
// → Cancel 시 OnCompleted 발행 → 자동 구독 해제
```

#### 성능 순위

| 방법 | 특성 |
|------|------|
| `Disposable.Combine(d1, d2, ...)` | 정적 개수, 최적 |
| `Disposable.CreateBuilder()` | 동적, 빌드 시 최적 |
| `DisposableBag` | 구조체, 간단 |
| `CompositeDisposable` | 클래스, Remove 지원, 가장 느림 |

### Observable Tracker (디버깅)

`Window > Observable Tracker`에서 활성 구독 모니터링. 구독 누수 찾을 때만 활성화.

## UniRx와의 주요 이름 변경

| UniRx | R3 |
|-------|-----|
| `Throttle` | `Debounce` |
| `Sample` | `ThrottleLast` |
| `Buffer` | `Chunk` |
| `BatchFrame` | `ChunkFrame` |
| `StartWith` | `Prepend` |
| `Amb` | `Race` |
| `Finally` | `Do(onDispose:)` |
| `.Value` (ReadOnly) | `.CurrentValue` |
