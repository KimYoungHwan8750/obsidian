```json
"query": {
	"hybrid": {
		"queries": [
			"bool": {
			  "should": [
			    { "rank_feature": { "field": "content_sparse", "query": "token_1", "boost": 1.5 } },
			    { "rank_feature": { "field": "content_sparse", "query": "token_2", "boost": 0.8 } },
			  ]
			}
		]
	}
}
```

| **요소**       | **논리적 의미** | **특징**                                                                       |
| -------------- | --------------- | ------------------------------------------------------------------------------ |
| **`must`**     | **AND**         | 이 조건은 **무조건 만족**해야 함. 점수에 영향을 줌.                            |
| **`filter`**   | **AND**         | 무조건 만족해야 하지만, **점수 계산은 안 함**. (속도 빠름, 예: 날짜, 카테고리) |
| **`should`**   | **OR**          | 있으면 좋고 없어도 되지만, **있으면 점수가 올라감**. (가중치 부여용)           |
| **`must_not`** | **NOT**         | 이 조건이 포함된 문서는 **무조건 제외**.                                       |

## filter, must 차이
- **`filter` (자격 요건)**: "전공이 '컴퓨터공학'인가?"
    
    - 이건 점수를 매길 필요가 없다. 맞으면 합격, 아니면 탈락입니다. 결과가 명확하니 속도가 매우 빠르고, 한 번 검사한 결과는 **캐시(Cache)**에 저장해서 재사용한다.
        
- **`must` (자기소개서 점수)**: "자기소개서에 '열정'이라는 단어가 얼마나 간절하게 들어있는가?"
    
    - 똑같이 '열정'이 들어있어도 1번 들어간 사람보다 10번 들어간 사람의 점수를 더 높게 책정할 수 있다. 이때는 유사도 점수를 계산해야 하므로 `must`를 쓰면 된다.
