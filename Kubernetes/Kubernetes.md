## Kubernetes란?
보통 줄여서 k8s로 지칭한다.
**스왑(Swap) 비활성화:** 쿠버네티스는 스왑 메모리가 켜져 있으면 작동하지 않음. (`sudo swapoff -a`)


**Liveness Probe & Readiness Probe:** * 컨테이너가 살아있는지, 손님을 받을 준비가 됐는지 체크하는 기능입니다.

- **Deployment**: **가장 많이 쓰는 Kind.** 앱의 원하는 상태(개수, 버전 등)를 정의하고 자동으로 배포/관리해 줍니다.
    
- **StatefulSet**: DB처럼 **순서와 고유 ID, 상태**가 중요한 앱을 위해 사용합니다. (Pod 이름이 `db-0`, `db-1` 식으로 고정됨)
    
- **Job / CronJob**: 한 번 실행되고 끝나는 작업(Batch)이나, 정해진 시간마다 반복되는 작업(예: 새벽 3시 백업)에 사용합니다.
- **Service**: 여러 개의 Pod을 하나로 묶어주는 **고정된 주소(전화번호)**입니다. Pod이 죽었다 살아나서 IP가 바뀌어도 Service 주소는 변하지 않습니다.
    
- **Ingress**: 클러스터의 **중앙 현관문.** 도메인(`example.com`)이나 URL 경로(`/api`)에 따라 트래픽을 적절한 Service로 배달해 줍니다. (L7 로드밸런서)
- **ConfigMap**: 일반적인 설정값(환경 변수, 설정 파일 내용 등)을 저장합니다.
    
- **Secret**: 비밀번호, API 키, SSL 인증서 같이 **보안이 필요한 데이터**를 저장합니다. (메모리에 저장되어 더 안전함)
- **PersistentVolume (PV)**: "이 컴퓨터에 100GB짜리 D드라이브 공간이 있다"고 등록하는 **실제 저장 공간.**
    
- **PersistentVolumeClaim (PVC)**: "나는 10GB의 공간이 필요해!"라고 앱이 요청하는 **신청서.**
    
- **StorageClass**: 저장 공간을 자동으로 생성해주는 **설계도.** (클라우드 환경에서 주로 사용)

## 5. 기타 필수 요소

- **Namespace**: 클러스터 안에서 구역을 나누는 **가상 파티션.** (예: `dev`, `prod` 팀별/환경별 구분)
    
- **Node**: Pod이 실제로 돌아가는 **물리적/가상 서버 그 자체.**

|**카테고리**|**Kind**|**비유**|
|---|---|---|
|**관리**|Deployment|팀장 (직원들이 일하게 시킴)|
|**통신**|Service|내선 번호 (담당자에게 연결)|
|**입구**|Ingress|안내 데스크 (방문객 안내)|
|**설정**|ConfigMap|게시판 (공지사항/설정 전달)|
|**보안**|Secret|금고 (비밀번호 보관)|
|**저장**|PVC|땅 문서 (내 자리 찜하기)|
