### 1. 언어 및 환경 설정 (Setup 시리즈)

코드를 가져온 뒤, 해당 언어를 실행할 수 있는 환경을 만드는 액션들입니다. 특정 버전을 지정할 수 있어 팀원 모두 동일한 환경에서 빌드할 수 있게 해줍니다.

- **`actions/setup-node@v4`**: Node.js 환경 구축. npm 캐싱 기능도 내장되어 있어 빌드 속도를 높여줍니다.
    
- **`actions/setup-python@v5`**: Python 인터프리터 설치 및 pip 환경 설정.
    
- **`actions/setup-java@v4`**: JDK(Java Development Kit) 설치. (Temurin, Zulu 등 배포판 선택 가능)
    
- **`actions/setup-go@v5`**: Go 언어 환경 설정

3. 속도 최적화 (Caching)랑 같이 쓰면 좋다. node modules, packages 캐싱
매번 수백 개의 라이브러리를 새로 다운로드하면 시간이 너무 오래 걸립니다. 이를 방지하는 액션입니다.

actions/cache@v4: 의존성 파일(node_modules, pip packages 등)을 저장해 두었다가, 파일 내용에 변화가 없으면 다음 실행 때 그대로 재사용합니다.

효과: 빌드 시간을 절반 이하로 단축할 수 있습니다.
