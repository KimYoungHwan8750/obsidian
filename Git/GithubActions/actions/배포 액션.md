2. 결과물 저장 및 공유 (Artifacts 시리즈)
빌드 단계에서 만든 파일(예: .apk, .zip, dist 폴더)을 저장하거나, 다음 작업(Job)으로 넘겨줄 때 사용합니다.

actions/upload-artifact@v4: 빌드가 끝난 후 결과물을 깃허브 서버에 업로드합니다. (워크플로우 실행 상세 페이지에서 다운로드 가능하게 됨)

actions/download-artifact@v4: 업로드된 결과물을 다른 작업(Job)이나 단계에서 다시 내려받습니다.

활용 예: '빌드 작업'에서 만든 파일을 '배포 작업'으로 전달할 때 필수입니다.
