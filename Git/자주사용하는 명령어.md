`git clean -rf`: Untracked 상태인 파일 삭제
`git reset head^`: 한 단계 전 커밋으로 복구 (기본 동작 soft reset)
`git reset --hard head^`: 한 단계 전 커밋으로 복구 (기본 동작 soft reset)
`git reflog`: 커밋 취소를 취소할 때 자주 사용함
`git reset --hard head && git clean -fd`
`git rm -r --cached .`: r옵션은 재귀란 뜻으로 하위 폴더에도 모두 적용하겠다는 뜻이다. 현재 컨텍스트에 깃의 모든 추적 목록을 제거하는 명령어. 파일은 보존되며 깃의 추적 이력만 지운다. 이 명령을 실행하고 `git add .` 이후 `git commit`, `git push`를 진행하게 되면 모든 파일들이 새로 추가되는 것과 같이 동작한다.

head와 head^ 차이
현재 커밋을 하지 않은 상태로 변경 내역을 취소하려고 한다. 그렇다면 head를 사용해야하며, 실수로 커밋까지 한 상태에서 잘못으 인지하고 변경 내역을 취소하려고 하면 head^를 사용해야한다.


`git clean`:
	`fdn`: 드라이 런, 삭제될 파일 목록만 보여줌
	`fdi`: 대화형 쉘 제공, 삭제될 파일을 선택하거나 필터링 가능
